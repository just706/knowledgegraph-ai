## 0. 项目一句话定义

基于 LLM + RAG + 知识图谱的**智能学习助手**，帮助用户从"拥有资料"转变为"真正理解和掌握知识"，最终成长为一个 **AI 学习操作系统**（而非聊天机器人）。

核心闭环：资料输入 → AI 理解 → 知识组织 → 学习交互 → 练习反馈

## 1. 产品需求文档（PRD）

### 1.1 产品概述
面向学生和知识学习者的智能学习系统。通过大语言模型、知识库、知识图谱和智能 Agent 技术，将用户上传的学习资料转化为结构化知识体系，提供智能问答、思维导图、自动出题、错题分析等能力。

### 1.2 用户痛点
1. 学习资料数量多，难以整理。
2. 知识点之间缺少关联，无法形成体系。
3. 遇到问题需要反复搜索，学习效率低。
4. 缺少针对性的练习和反馈。
5. 不知道自己的薄弱知识点。

### 1.3 产品目标（MVP）
1. 实现个人知识库管理。
2. 实现基于资料的 AI 问答。
3. 自动生成知识结构。
4. 自动生成思维导图。
5. 根据知识点自动出题。
6. 建立错题反馈闭环。

### 1.4 目标用户
- 高校学生
- 技术学习者
- 考试备考用户
- 企业培训学习者

### 1.5 核心功能需求
| 模块 | 功能点 |
|------|--------|
| 用户系统 | 注册登录、个人学习空间、学习记录管理 |
| 知识库 | 上传 PDF/TXT/Markdown、文档解析、文档管理、知识摘要生成 |
| AI 问答 | 基于个人资料回答、提供知识来源、简单解释/深入分析模式 |
| 知识图谱 | 自动抽取知识节点、自动建立知识关系、图形化展示 |
| 思维导图 | 自动生成课程结构、节点展开查看、辅助学习导航 |
| 自动出题 | 根据知识点生成题目、提供答案和解析 |
| 错题本 | 保存错误记录、分析错误原因、推荐复习内容 |

### 1.6 核心用户流程
注册登录 → 创建学习空间 → 上传学习资料 → AI 解析资料 → 生成知识体系 → 查看知识图谱/思维导图 → AI 问答学习 → 自动练习 → 错题分析 → 持续优化学习路径

### 1.7 MVP 功能范围（优先级）
- **P0（必须）**：用户认证、文件上传、文档解析、RAG 问答
- **P1**：知识图谱生成、思维导图展示
- **P2**：自动出题、错题管理
- **暂不实现**：视频理解、多模态学习、商业化推荐系统

### 1.8 非功能需求
- 性能：问答响应时间合理；支持多用户数据隔离。
- 安全：用户数据权限控制；文件访问保护。
- 可维护性：模块化设计；API 文档完善；配置环境变量管理。

### 1.9 技术架构需求
- 前端：Vue3 + TypeScript + Element Plus
- 后端：FastAPI + Python
- AI：LLM + Embedding + RAG + Agent
- 数据库：PostgreSQL（业务数据）、Neo4j（知识图谱）、Chroma/pgvector（向量检索）
- 可视化：AntV G6、Vue Flow

### 1.10 产品核心指标（MVP 验证）
1. 用户能够成功上传并解析资料。
2. AI 能够基于资料回答问题。
3. 系统能够生成合理知识结构。
4. 自动生成题目质量可接受。
5. 用户能够通过错题发现知识薄弱点。

### 1.11 未来规划
- **V2**：个性化学习计划、学习能力评估、多 Agent 协作
- **V3**：AI 导师、视频课程理解、多模态学习、商业化平台

## 2. MVP 开发计划

### 2.1 项目目标
构建基于 LLM、RAG、知识图谱的智能学习助手，实现：
用户上传学习资料 → AI 解析 → 知识组织 → 智能问答 → 自动出题 → 错题反馈。

### 2.2 MVP 功能范围
1. 用户系统：注册、登录、个人学习空间
2. 知识库管理：PDF/TXT/Markdown 上传、文档解析、状态管理、自动摘要和知识点
3. RAG 智能问答：用户问题 → Embedding → 向量检索 → LLM 生成答案
4. 知识图谱生成：抽取知识点、建立关系、Neo4j 存储展示
5. 思维导图生成：树形学习地图、节点展开查看解释
6. AI 自动出题：单选/判断/简答，输出题目+答案+解析+知识点
7. 错题本：用户答案、正确答案、错误原因、关联知识点

### 2.3 技术架构
- 前端：Vue3 + TypeScript + Vite + Element Plus + Vue Flow + AntV G6
- 后端：FastAPI + Python + SQLAlchemy + Pydantic
- AI：LLM API + Embedding + RAG + LangChain/LangGraph
- 数据库：PostgreSQL（用户/题目/学习记录）、Neo4j（知识图谱）、Chroma（向量检索）

### 2.4 开发阶段
| Phase | 内容 |
|-------|------|
| Phase 1 | 项目初始化：前后端工程、开发环境、Git |
| Phase 2 | 用户系统：注册登录、JWT 认证、数据库设计 |
| Phase 3 | 知识库模块：文件上传、PDF 解析、文本切片、向量存储 |
| Phase 4 | RAG 系统：Embedding、检索、AI 问答接口 |
| Phase 5 | 知识图谱：实体抽取、关系抽取、Neo4j 存储、图谱展示 |
| Phase 6 | 前端展示：知识库页、AI 聊天页、图谱页、思维导图页 |
| Phase 7 | AI 出题：自动生成题目、答案解析 |
| Phase 8 | 错题系统：错题保存、错题分析、学习建议 |

### 2.5 Agent 执行规则
1. 按 Phase 顺序开发。
2. 每完成一个阶段输出：修改文件列表、已实现功能、测试结果、下一步计划。
3. 所有配置使用环境变量。
4. API 必须提供 Swagger 文档。
5. 优先保证 MVP 功能，不提前复杂化架构。

### 2.6 MVP 验收标准（必须完成）
- 用户注册登录
- 文件上传
- AI 文档解析
- RAG 问答
- 知识图谱生成
- 思维导图展示
- AI 自动出题
- 错题记录分析

### 2.7 后续扩展
- V2：学习计划 Agent、能力评估、多 Agent 协作
- V3：视频理解、多模态学习、AI 导师系统

## 3. AI 宪法 v1.0

> 适用对象：AI Coding Agent、自动化开发 Agent、项目维护 Agent

### 第一章 总则（Project Mission）
构建基于 LLM、RAG、知识图谱技术的智能学习助手，实现从知识获取、知识组织、知识理解到学习反馈的完整闭环。
核心流程：资料输入 → AI 理解 → 知识组织 → 学习交互 → 练习反馈

### 第二章 MVP 开发原则
1. MVP 优先：先实现核心学习闭环，再扩展高级能力。
2. 用户价值优先：所有功能必须服务于学习效率提升。
3. AI 增强原则：AI 负责理解、总结、推理和生成，程序负责业务流程、数据和权限。
4. 简洁架构原则：避免为了技术复杂度引入无价值组件。

### 第三章 Agent 代码行为规范
1. 修改代码前必须分析现有架构和依赖关系。
2. 禁止直接覆盖已有核心代码。
3. 技术替换必须说明原因、收益、风险和迁移成本。
4. 保持模块化设计，避免单文件过度膨胀。

### 第四章 AI 系统设计规范
1. 知识问答必须优先采用 RAG 流程。
2. AI 回答应尽可能提供知识来源和关联信息。
3. 知识图谱必须明确区分节点和关系。
4. Prompt 必须独立管理，不允许大量硬编码。

### 第五章 数据规范
1. 用户数据必须隔离并保证安全。
2. 数据库结构修改必须通过迁移管理。
3. 知识来源优先级：用户上传资料 > 用户笔记 > 系统知识库 > 外部信息。

### 第六章 前端开发规范
1. 页面设计以简单、清晰、易学习为目标。
2. 所有异步任务必须提供状态反馈。
3. 核心页面包括：首页、知识库、AI 问答、知识图谱、思维导图、错题本。

### 第七章 测试规范
1. 每个模块必须经过接口测试。
2. AI 功能必须测试异常输入、空输入、长文本和模型失败情况。
3. 每次迭代必须保证已有功能不被破坏。

### 第八章 技术决策原则
技术选择优先级：简单方案 > 成熟方案 > 可扩展方案 > 高级方案。
MVP 阶段禁止：微服务、Kubernetes、复杂分布式架构，除非明确业务需要。

### 第九章 最终目标
本项目不是简单的 AI 聊天机器人，而是构建帮助用户建立个人知识体系、持续优化学习过程的 AI 学习操作系统。

### Agent 最高优先级规则
1. 不破坏已有功能。
2. MVP 优先。
3. 用户价值优先。
4. AI 负责智能，代码负责业务。
5. 保持架构简单。
6. 每一步都可运行验证。
7. 文档与代码同步。
